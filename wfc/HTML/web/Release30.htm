Release 30

Bug Fixes/Enhancements

CCryptographicAlgorithm - Added more enums for the new algorithms included in VC5.

CCryptographicKey - Fixed a design problem in the Crypto API. Now what you decrypt what you encrypted. This ain't what happens when you use the Crypto API. In short, I added automatic stripping of any pad bytes.

CRandomNumberGenerator - Guy Gascoigne - Piggford (ggp@informix.com) provided a better random number generator algorithm. This one is good for 900 million pseudo-random numbers per seed before it starts to repeat.

New Classes

CDataChunk - Allows for handling of data chunks. Patterned after RIFF.
CDataFile - A derivative of CFile that makes it easy to read/write CDataChunk's.
CDataSocket - A derivative of CDataFile that allows you to transmit CDataChunk's over a CSimpleSocket.
CPasswordChangeFilter - A class that makes it easy to intercept password changes.
CQueue - A simple thread-safe data queue. Designed for speed (i.e. no linked list).
CReedSolomonErrorCorrectionCode - Allows you to put a forward error correction code onto a block of data. This class blocks data into 255 byte chunks. Up to 16 of those bytes can be scrambled and the errors will automatically be fixed.
